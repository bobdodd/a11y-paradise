# Contributing to a11y-paradise

Thank you for your interest in contributing. This project aims to make accessibility knowledge and tools freely available, and contributions from the community help make that possible.

## What You Can Contribute

- **Training content**: New guides, tutorials, examples, or improvements to existing materials
- **Tools**: Accessibility testing utilities, auditing scripts, or developer helpers
- **Resources**: Links to valuable references, research papers, or community resources
- **Corrections**: Fixes for errors in content, code, or outdated information

## How to Contribute

1. **Open an issue first** — Before starting work, open an issue describing what you'd like to add or change. This helps avoid duplicate effort and ensures alignment with the project's goals.
2. **Fork the repository** and create a branch for your changes.
3. **Follow existing conventions** — Match the style and structure of existing content.
4. **Submit a pull request** — Reference the related issue and describe your changes clearly.

## Content Guidelines

- Write for a technical audience (developers and accessibility professionals)
- Use plain, direct language
- Include practical examples wherever possible
- Cite sources and standards (e.g., WCAG success criteria) when referencing requirements
- Ensure all content you contribute is your own work or properly attributed

## Accessibility

This is an accessibility project. All contributions should themselves be accessible:

- Use semantic Markdown structure (proper headings, lists, link text)
- Provide alt text for any images
- Ensure code examples demonstrate accessible patterns

## Questions?

Open an issue or reach out via GitHub.
